# 小红书收藏夹 AI 知识点总结 —— 方案设计

## 1. 你的需求拆解

- 小红书有一个收藏夹，里面有上百条关于 AI / LLM / Agent 的技术分享帖子。
- 逐条看太慢，且帖子质量参差（有的已会、有的重点不突出、有的质量低）。
- 希望用大模型自动总结，输出一份**知识点总结**。
- 总结里每个知识点都能**快速定位到原始帖子**，方便回看原文。

## 2. 核心约束（先说清楚，决定了方案形态）

| 约束 | 说明 | 应对 |
|------|------|------|
| 小红书反爬 | 收藏夹数据需登录态，批量请求易触发风控 | 半自动：你本地登录一次，脚本复用登录态，控制频率 |
| 对话模型不能联网批量跑 | 当前对话的我无法主动爬数据、无法循环调外部 API | 拆成两段：脚本负责取数据，我负责做总结 |
| 图片型干货 | 不少知识点写在图片里 | 抓取时下载图片 → OCR 转文字 |
| 溯源 | 每个知识点要能回到原帖 | 全流程保留 `note_id` / 原帖链接，贯穿到最终输出 |

## 3. 整体数据流

```
┌─────────────────────────────────────────────────────────────┐
│  阶段一：数据获取（脚本，在你本地跑）                          │
│                                                              │
│  小红书登录(你扫码) → 收集收藏夹全部帖子ID → 逐条点击卡片抓取 │
│        → 下载图片 → 大模型/OCR 提取图片文字                    │
│        → 输出 data/notes/*.json （每条帖子一个结构化文件）    │
└─────────────────────────────────────────────────────────────┘
（注：抓取的关键实现——点击 a.cover 拿 token、虚拟滚动收集 ID 等，
  详见 README 第九节「抓取实现原理与踩坑记录」，本文档为早期设计，以 README 为准）
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  阶段二：内容总结（我，在对话里处理）                          │
│                                                              │
│  读取 data/notes/*.json → 按提示词模板逐条/分批提炼知识点    │
│        → 去重、按主题归类、标注质量                           │
│        → 输出 output/summary.json（结构化，带 note_id 溯源）  │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  阶段三：成果呈现                                            │
│                                                              │
│  summary.json →  ① 知识点总结.md （按主题分章，附原帖链接）  │
│                  ② output/index.html（可按主题/质量筛选浏览，│
│                     每个知识点可点击跳转原帖）                │
│                  ③ 可选：publish.py 自动发布到 GitHub Pages   │
└─────────────────────────────────────────────────────────────┘
```

## 5. 每条帖子的结构化格式（data/notes/{note_id}.json）

```json
{
  "note_id": "65xxxxxxxxxxxxxx",
  "url": "https://www.xiaohongshu.com/explore/65xxxxxxxxxxxxxx",
  "title": "帖子标题",
  "author": "作者昵称",
  "publish_time": "2025-xx-xx",
  "text": "帖子正文文字",
  "image_ocr": ["图1里识别出的文字", "图2里识别出的文字"],
  "images": ["data/images/65xxx_0.jpg", "..."],
  "tags": ["#LLM", "#Agent"]
}
```

## 6. 最终知识点的结构化格式（output/summary.json）

```json
{
  "topics": [
    {
      "topic": "RAG 检索增强",
      "points": [
        {
          "point": "知识点的精炼描述（一句话讲清重点）",
          "detail": "稍展开的说明 / 关键结论",
          "quality": "high | medium | low",
          "source_note_id": "65xxxxxxxxxxxxxx",
          "source_url": "https://www.xiaohongshu.com/explore/65xxx",
          "source_title": "原帖标题"
        }
      ]
    }
  ]
}
```

> 溯源关键：每个知识点都带 `source_note_id` + `source_url` + `source_title`，
> 在 Markdown 和网页里都会渲染成可点击链接，点一下就能回到原帖。

## 7. 大模型总结的策略（保证质量 + 溯源准确）

1. **逐条提炼**：每条帖子单独喂给模型，让它输出「该帖包含哪些知识点 + 质量评级」，
   强制带上 `note_id`，从源头保证溯源不串。
2. **质量过滤**：对 `low` 质量、纯营销/水帖打标，最终可在网页里一键隐藏。
3. **跨帖去重归类**：所有知识点汇总后，按主题聚类，重复知识点合并但**保留所有来源链接**
   （这样你能看到「这个点有几篇帖子都讲过」）。
4. **不臆造**：提示词中明确要求只总结帖子里出现的内容，不补充模型自己的知识，避免污染。

## 8. 你需要参与的步骤（其余全自动）

1. 跑抓取脚本时，**扫码登录**一次小红书。
2. 把收藏夹链接/ID 填进 `config.json`。
3. 抓取 + OCR 完成后，把 `data/notes/` 的内容交给我（对话里），我来做总结。
4. 拿到 `知识点总结.md` 和 `web/index.html`，开始高效学习。
