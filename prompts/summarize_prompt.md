# 大模型总结提示词模板

本文件提供两段提示词：
- **提示词 A（逐条/分批提炼）**：把帖子内容喂给模型，提炼知识点 + 质量评级，强制溯源。
- **提示词 B（汇总归类去重）**：把 A 的所有结果合并，按主题聚类、合并重复、保留所有来源。

> 推荐用法：把 `data/notes/*.json` 分批（每批 5~10 条）贴进对话，配合提示词 A 得到中间结果；
> 全部跑完后用提示词 B 做一次汇总，得到最终 `output/summary.json`。

---

## 提示词 A：逐条提炼知识点

```
你是一个严谨的技术知识提炼助手。下面是若干条小红书技术帖子的内容（关于 AI / LLM / Agent）。
请为每一条帖子提炼其中的核心知识点。

【严格要求】
1. 只总结帖子里**实际出现**的内容，不要补充你自己的知识，不要臆造、不要发挥。
2. 每个知识点用一句话讲清重点（point），再用 1~3 句话给出关键说明/结论（detail）。
3. 给每条帖子整体打一个质量评级 quality：
   - high：有干货、有具体方法/结论/数据，值得细看
   - medium：有一定信息量但不突出
   - low：内容空泛 / 纯营销引流 / 重复常识 / 无实质信息
4. 每个知识点必须带上来源字段：source_note_id / source_url / source_title，
   直接取自我给你的帖子数据，**不得修改或编造**。
5. 若某条帖子没有任何有价值知识点（如纯广告），points 输出空数组，quality 标 low。

【输出格式】严格输出 JSON 数组，每个元素对应一条帖子：
[
  {
    "source_note_id": "...",
    "source_url": "...",
    "source_title": "...",
    "quality": "high|medium|low",
    "topic_hint": "你判断该帖最贴近的主题，如 RAG/Agent/微调/Prompt工程/向量数据库/多模态/工程部署/评测 等",
    "points": [
      { "point": "一句话重点", "detail": "关键说明/结论" }
    ]
  }
]

【帖子数据】
<在此粘贴本批次帖子的 JSON，每条包含 note_id/url/title/text/image_ocr 等>
```

## 提示词 B：汇总、按主题归类、去重

```
下面是我对一批小红书技术帖子逐条提炼出的知识点（JSON 数组）。请你做汇总整理：

【要求】
1. 按主题（topic）对所有知识点重新聚类归并，主题示例：
   Prompt 工程 / RAG 检索增强 / Agent 设计 / 模型微调与训练 / 向量数据库 /
   多模态 / 推理优化与部署 / 评测与可观测 / 工具与框架 / 行业应用 / 其他
2. 合并**语义重复**的知识点，但合并时**保留所有来源**（多个 source 链接都列出），
   这样我能看到哪些点被多篇帖子反复强调。
3. 保留每个知识点的质量评级（取来源中最高者）。
4. 不要丢失任何 high/medium 知识点；low 质量的可单独归到“低质量/可跳过”分组。
5. 主题内按重要性（被提及次数、质量）从高到低排序。

【输出格式】严格输出如下 JSON（即最终 output/summary.json）：
{
  "topics": [
    {
      "topic": "主题名",
      "points": [
        {
          "point": "一句话重点",
          "detail": "关键说明/结论",
          "quality": "high|medium|low",
          "sources": [
            { "note_id": "...", "url": "...", "title": "..." }
          ]
        }
      ]
    }
  ]
}

【逐条提炼结果】
<在此粘贴提示词 A 产出的所有 JSON 结果>
```

## 给“当前对话模型（我）”的简化用法

如果你不想自己跑两段提示词，最简单的方式是：
> 在对话里直接说：“开始总结”，并把 `data/notes/` 下的 JSON 内容贴给我（或告诉我目录已就绪），
> 我会自动按上面的策略处理，最终生成 `output/summary.json`、`output/知识点总结.md`，
> 并刷新 `web/index.html` 的数据。
